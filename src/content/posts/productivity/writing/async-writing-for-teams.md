## 세 가지 원칙

1. **맥락 우선**: 문제 배경을 먼저 정리하면 질문 수가 줄어듭니다.
2. **명확한 요청**: 승인, 피드백, 정보 공유 등을 분리합니다.
3. **기한 정의**: `Need feedback by Fri`처럼 명시적으로 적습니다.

## 템플릿 예시

```
# TL;DR

# Background

# Options

# Decision
```

## 리뷰 플로우

- PR처럼 Reviewer를 지정
- 댓글은 스레드로 유지
- Slack에는 링크 + 데드라인만 공유

## 도구 추천

- Notion AI의 요약 기능으로 TL;DR 자동화
- GitHub Discussions로 기술 RFC 관리

문서 기반 루틴이 자리 잡히면 시차가 있는 팀에서도 일관된 속도를 낼 수 있습니다.
